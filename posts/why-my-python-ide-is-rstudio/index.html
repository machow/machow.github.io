Why my Python IDE is RStudio

January 3, 2019 by Michael Chow

Intro

In this post, I’ll cover what led me to switch to RStudio for all my python data science work. It boils down to three things:

  1. Rmarkdown is a great format (simple text, incl support for params)
  2. The reticulate package is simple, but powerful
  3. Rstudio makes it quick to open project specific sessions (4.) All these things could exist w/ Jupyter notebooks

context:

  • been programming python for over a decade
  • developed / maintained critical python libraries for DataCamp
  • as a software engineer, vim + ipython worked great. For DS, need Rstudio.

(caveat: I’m most interested in being able to generate static reports)
